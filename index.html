HTML

No. Elemento Proposito Ejemplo
1. div Elemento generico

<div> Texto </div>

2. span Se puede usar para agrupar elementos con fines de estilo (usando los atributos class o id), o porque comparten valores de atributo, como lang.

<span style="color: red;">texto en rojo</span>

3. header descrive contenido destinado a la cabesera de la pagina

<header>
  <h1>Bienvenidos a mi sitio web</h1>
  <nav>Menú de navegación</nav>
</header>

4. article encierra contenido independiente del resto de la pagina web

<article>
  <h2>Título del artículo</h2>
  <p>Este es un párrafo dentro de un artículo.</p>
</article>

5. section se utiliza para indicar contenido que típicamente cubre un tema específico. Puede ser usado para dividir el contenido dentro de un elemento article.

<section>
  <h2>Sección 1</h2>
  <p>Contenido relevante de la sección.</p>
</section>

6. nav representa una sección de una página cuyo propósito es proporcionar enlaces de navegación, ya sea dentro del mismo documento o hacia otros documentos.

<nav>
  <ul>
    <li><a href="#home">Inicio</a></li>
    <li><a href="#about">Acerca de</a></li>
  </ul>
</nav>

7. aside representa una parte de un documento cuyo contenido solo está indirectamente relacionado con el contenido que lo rodea.

<aside>
  <h3>Artículos recomendados</h3>
  <ul>
    <li>Artículo 1</li>
    <li>Artículo 2</li>
  </ul>
</aside>

8. footer representa un pie de página para su ancestro de la categoría SECTIONING más cercano (article, aside, nav, section), o para el elemento body si no existe tal ancestro.

<footer>
  <p>© 2025 Mi Sitio Web. Todos los derechos reservados.</p>
  <nav>
    <a href="#privacy">Política de privacidad</a>
    <a href="#terms">Términos de servicio</a>
  </nav>
</footer>

9. main representa los contenidos dominantes del documento. Un documento no debe tener más de un elemento main. Los elementos padres aceptados son html, body, div y form.

<main>
  <h1>Bienvenidos a nuestra página principal</h1>
  <p>Aquí se encuentra el contenido más importante de nuestro sitio web.</p>
</main>

10. h1, h2, h3, h4, h5, h6 El contenido de los elementos se muestra en diferentes tamaños y en negrita.

<h1>Este es el encabezado principal</h1>
<h2>Este es un subtítulo de segundo nivel</h2>
<h3>Subtítulo de tercer nivel</h3>
<h4>Subtítulo de cuarto nivel</h4>
<h5>Subtítulo de quinto nivel</h5>
<h6>Subtítulo de sexto nivel</h6>

11. p Los párrafos se definen con el elemento p . Los párrafos se utilizan para dividir un texto largo en bloques de texto más pequeños, más fáciles de digerir y que hablan normalmente de una sola idea.

<p>Este es un párrafo de texto dentro de un documento HTML.</p>

12. br El elemento vacío de salto de línea br hace que el navegador avance a la siguiente línea antes de mostrar el siguiente elemento o porción de texto en una página web.

<p>Este es el primer párrafo.<br>Este es el segundo párrafo, después del salto de línea.</p>

13. hr El elemento hr representa una transición entre párrafos a un tema separado, pero relacionado.

<p>Texto antes de la línea.</p>
<hr>
<p>Texto después de la línea.</p>

14. wbr El elemento vacío wbr le indica al navegador donde podría razonablemente insertar un salto de línea para ajustar el contenido que es más grande que la ventana actual del navegador. Es el navegador el que toma la decisión de si realmente se utiliza o no un salto de línea.

<p>Este es un ejemplo de una palabra muy larga:<wbr>Supercalifragilisticexpialidocious</p>

15. sup El elemento sup permite definir superíndices.

<p>La fórmula de la energía es E = mc<sup>2</sup></p>

16. sub El elemento sub permite definir subíndices.

<p>La fórmula del agua es H<sub>2</sub>O</p>

17. strong indica que su contenido tiene una gran importancia, seriedad o urgencia. Los navegadores normalmente muestran el contenido en negrita.

<p>Este es un texto con <strong>énfasis fuerte</strong></p>

18. em Define un texto con énfasis, generalmente se muestra en cursiva

<p>Este es un texto con <em>énfasis</em></p>

19. mark permite resaltar texto de interés.

<p>Este es un <mark>texto resaltado</mark></p>

20. q indica que el texto encerrado es una cita corta en línea.

<p>Él dijo: <q>La vida es bella</q></p>

21. blackquote El elemento blockquote indica que el texto contenido es una cita extensa.

<blockquote>
  Este es un bloque de cita largo, que puede abarcar varios párrafos.
  <p>Aquí hay más texto dentro de la cita.</p>
</blockquote>

22. cite permite incluir el nombre de un trabajo (Un libro, un ensayo, un poema, una película, una pintura, un programa de computadora, etc.). Este puede ser un trabajo que se cita en la pagina, o puede ser simplemente un trabajo que se menciona de pasada.

<p>Este es un ejemplo de una cita: <cite>El Gran Libro</cite></p>

23. dfn se utiliza para identificar el término a definir, mientras que la definición del término se coloca fuera del elemento.

<p>El término <dfn>HTML</dfn> significa Lenguaje de Marcado de Hipertexto.</p>

24. abbr se utiliza para identificar una abreviatura.

<p>La abreviatura <abbr title="Cascading Style Sheets">CSS</abbr> se usa para hojas de estilo en cascada.</p>

25. small permite definir la letra chica y los comentarios secundarios de una página, como descargos de responsabilidad, derechos de autor, información sobre licencias y restricciones legales.

<p>Este es un texto en tamaño normal, pero este es un texto en <small>texto pequeño</small>.</p>

26. ins marca las partes de un texto que han sido añadidos al documento.

<p>Este es un texto original, y ahora se ha <ins>insertado un nuevo texto</ins>.</p>

27. del marca las partes de un texto que han sido eliminadas del documento.

<p>Este es un texto original, y ahora se ha <del>eliminado un texto</del>.</p>

28. s muestra el texto tachado con una línea que lo atraviesa.

<p>Este texto está <s>tachado</s> para mostrar que es incorrecto.</p>

29. code muestra su contenido con formato para indicar que el texto es un fragmento corto de código de computadora.

<p>El siguiente es un ejemplo de código: <code>let x = 10;</code></p>

30. samp se utiliza para encerrar texto que representa la salida de un programa de computadora (Los mensajes de la computadora al usuario).

<p>El resultado del programa es: <samp>Error 404: Página no encontrada</samp></p>

31. kbd representa un fragmento de texto en línea que indica la entrada del usuario desde un teclado (Aunque también se puede usar para representar otra entrada, como los comandos de voz).

<p>Presiona <kbd>Ctrl</kbd> + <kbd>C</kbd> para copiar.</p>

32. pre contiene texto que debe presentarse exactamente como está escrito en el archivo HTML.

<pre>
  Este es un bloque de texto preformateado.
  Se conservan los espacios y saltos de línea.
</pre>

33. address indica que el HTML contenido proporciona información de contacto para una persona o personas, o para una organización.

<address>
  <p>Correo electrónico: <a href="mailto:ejemplo@dominio.com">ejemplo@dominio.com</a></p>
  <p>Teléfono: (123) 456-7890</p>
</address>

34. caracteres especiales A veces, es necesario incluir caracteres HTML, como <, > y &, en una página. Para garantizar que el navegador no interprete los caracteres como código HTML, se utilizan character entity references o numeric character references. < = "&lt;"
> = "&gt;"
& = "&amp;"
